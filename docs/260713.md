## 1. 프로젝트 개요

개인 성장 기록을 한곳에 모아 관리할 수 있는 웹 서비스 **GrowLog** 개발을 시작하였다.

## 4. 프로젝트 구조 구상

Spring MVC 구조를 기준으로 프로젝트의 계층을 나누기로 하였다.

    ↓

    ↓

    ↓

각 계층의 역할은 다음과 같이 정리하였다.

### Repository

## 5. MEMBER 테이블 설계

프로젝트에서 가장 먼저 필요한 회원 기능을 구현하기 위해 `MEMBER` 테이블 구조를 설계하였다.

```sql
CREATE TABLE member (
    member_no BIGINT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    nickname VARCHAR(50) NOT NULL UNIQUE,
    user_name VARCHAR(50) NOT NULL,
    profile_image VARCHAR(255),
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
);
```

### 주요 컬럼

| 컬럼              | 설명                  |
| --------------- | ------------------- |
| `member_no`     | 회원 식별 번호            |
| `email`         | 로그인 아이디 및 이메일 인증 주소 |
| `password`      | 암호화된 비밀번호           |
| `nickname`      | 서비스에서 사용할 닉네임       |
| `user_name`     | 회원 이름               |
| `profile_image` | 프로필 이미지 경로          |
| `created_at`    | 회원가입 일시             |
| `updated_at`    | 회원정보 수정 일시          |

이메일과 닉네임은 중복되지 않도록 `UNIQUE` 제약조건을 적용하였다.

비밀번호는 원문이 아닌 암호화된 값을 저장할 예정이므로 최대 길이를 고려해 `VARCHAR(255)`로 설정하였다.

## 6. AWS RDS 및 Spring Boot 연결 설정

GrowLog의 데이터를 로컬 DB가 아닌 AWS RDS MySQL에 저장하기 위한 환경을 구성하였다.

`application.yaml`에 Datasource 연결 정보를 작성하였다.

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://RDS-ENDPOINT:3306/growlog?serverTimezone=Asia/Seoul&characterEncoding=UTF-8&allowPublicKeyRetrieval=true&useSSL=false
    username: DB 사용자명
    password: DB 비밀번호
```

RDS 연결을 위해 다음 항목도 확인하였다.

* RDS 퍼블릭 액세스 허용
* 보안 그룹의 MySQL 포트 `3306` 인바운드 규칙
* RDS 엔드포인트
* 데이터베이스명 `growlog`
* DB 사용자명 및 비밀번호
* JDBC 연결 옵션

DBeaver를 이용해 RDS의 `growlog` 데이터베이스에 접속하고 테이블 생성 여부를 확인하였다.

```sql
USE growlog;

SHOW TABLES;
```

## 8. 로그인 및 세션 처리 구조 설계

로그인에 성공한 회원 정보를 `HttpSession`에 저장하여 로그인 상태를 유지하기로 하였다.

```java
session.setAttribute("loginMember", member);
```

세션 Attribute 이름은 다음과 같이 통일하였다.

```text
loginMember
```

Home 페이지는 회원 전용 화면이므로 세션에 로그인 회원 정보가 없으면 로그인 페이지로 이동하도록 구성하였다.

```java
@GetMapping("/home")
public String homePage(HttpSession session) {

    Member loginMember =
            (Member) session.getAttribute("loginMember");

    if (loginMember == null) {
        return "redirect:/login";
    }

    return "home";
}
```

이를 통해 URL을 직접 입력하더라도 비로그인 사용자가 Home 페이지에 접근하지 못하도록 기본적인 접근 제어 구조를 만들었다.

## 14. 오늘 배운 점

* 프로젝트를 시작할 때는 화면부터 만드는 것보다 핵심 기능과 데이터 구조를 먼저 정리하는 것이 중요하다.
* 회원 기능은 이후 모든 데이터의 소유자를 구분하는 기준이 되므로 가장 먼저 구현하는 것이 적절하다.
* Spring Boot에서 원격 DB를 사용할 때는 애플리케이션 설정뿐 아니라 AWS 보안 그룹과 RDS 네트워크 설정도 함께 확인해야 한다.
* Entity를 화면에 직접 사용하기보다 요청 목적에 맞는 DTO를 분리하면 구조를 관리하기 쉽다.
* 로그인 성공 여부뿐 아니라 세션 저장과 회원 전용 페이지의 접근 제어까지 하나의 인증 흐름으로 설계해야 한다.
* 프로젝트 초기부터 파일과 패키지의 역할을 분리하면 기능이 늘어났을 때 유지보수가 쉬워진다.
