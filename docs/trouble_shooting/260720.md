# 📅 Troubleshooting - 2026.07.20

## 1. Goal 페이지 접근 시 로그인 세션 검증 추가

### 📌 문제

Goal 기능을 구현하면서 URL(`/goal`)을 직접 입력하면 로그인 여부와 관계없이 Goal 페이지에 접근할 수 있는 문제가 있었다.

회원 전용 기능임에도 인증되지 않은 사용자가 접근할 수 있는 구조였기 때문에 보안 및 데이터 접근 측면에서 문제가 발생할 수 있었다.

---

### 🔍 원인

GoalController에서 로그인 여부를 확인하지 않고 바로 목표 목록을 조회하도록 구현되어 있었다.

```java
@GetMapping
public String goalList(...) {
    ...
}
```

로그인 세션(`loginMember`)이 존재하는지 확인하는 과정이 없었기 때문에 비로그인 사용자도 Controller가 실행되었다.

---

### 🛠 해결 방법

GoalController에서 가장 먼저 세션에 저장된 로그인 정보를 확인하도록 수정하였다.

```java
Member loginMember = getLoginMember(session);

if (loginMember == null) {
    return "redirect:/login";
}
```

로그인 정보가 없는 경우 로그인 페이지로 이동하도록 처리하여 인증되지 않은 접근을 차단하였다.

---

### ✅ 결과

* 비로그인 사용자는 Goal 페이지 접근 시 로그인 페이지로 이동
* 로그인한 사용자만 Goal 기능 이용 가능
* 회원 전용 기능에 대한 접근 제어가 정상적으로 동작

---

### 💡 배운 점

* Controller에서는 비즈니스 로직보다 **인증 및 권한 확인을 우선적으로 수행하는 것이 중요하다.**
* 세션 기반 로그인에서는 모든 회원 전용 기능에서 로그인 여부를 먼저 검증해야 한다.

---

## 2. 로그인한 사용자만 자신의 Goal 조회하도록 수정

### 📌 문제

Goal 목록을 조회할 때 어떤 회원의 목표를 조회해야 하는지에 대한 기준이 필요했다.

회원마다 목표가 다르기 때문에 전체 데이터를 조회할 경우 다른 회원의 목표까지 함께 조회될 수 있는 문제가 있었다.

---

### 🔍 원인

초기에는 Goal 조회 시 로그인한 회원 정보를 기준으로 조회하는 로직이 명확하게 적용되어 있지 않았다.

Goal 테이블은 여러 회원이 함께 사용하는 데이터이므로 사용자 구분 없이 조회하면 데이터가 섞일 가능성이 있었다.

---

### 🛠 해결 방법

세션에 저장된 로그인 회원 정보를 가져와 회원 번호(`memberNo`)를 기준으로 목표를 조회하도록 수정하였다.

```java
Member loginMember = getLoginMember(session);

List<Goal> goalList =
        goalService.findGoalsByMember(loginMember.getMemberNo());
```

Service에서는 전달받은 회원 번호를 이용하여 해당 회원의 목표만 조회하도록 구현하였다.

---

### ✅ 결과

* 로그인한 회원의 목표만 조회
* 다른 회원의 데이터가 노출되지 않음
* 회원별 데이터 분리가 정상적으로 이루어짐

---

### 💡 배운 점

* 회원 데이터를 조회할 때는 **세션의 로그인 정보를 기준으로 조회하는 습관이 중요하다.**
* 다중 사용자를 지원하는 서비스에서는 항상 사용자 식별 정보를 기준으로 데이터를 조회해야 데이터 보안과 무결성을 유지할 수 있다.
