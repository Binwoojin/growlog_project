# 📅 Troubleshooting - 2026.07.15

## 1. 회원가입 시 `USER_NAME`에 null이 저장되는 오류

### 📌 문제

회원가입을 실행했을 때 다음과 같은 데이터 저장 오류가 발생하였다.

```text
DataIntegrityViolationException
Column 'USER_NAME' cannot be null
```

회원가입 화면에서 이름을 정상적으로 입력했지만, `MEMBER` 테이블의 `USER_NAME` 컬럼에는 값이 전달되지 않았다.

---

### 🔍 원인

회원가입 요청을 전달하는 DTO와 Member Entity의 필드명이 일치하지 않았다.

회원가입 화면에서는 사용자의 이름을 전송하고 있었지만 DTO, Service 또는 Entity로 값을 옮기는 과정에서 필드가 누락되거나 서로 다른 이름으로 작성되어 있었다.

예를 들어 다음과 같이 이름이 일치하지 않으면 값이 정상적으로 전달되지 않는다.

```java
// DTO
private String name;

// Entity
private String userName;
```

Spring의 데이터 바인딩과 객체 변환 과정에서는 각 계층에서 사용하는 필드명을 정확하게 연결해야 한다.

---

### 🛠 해결 방법

회원가입 DTO에 `userName` 필드를 추가하고, 입력 폼의 `name` 속성도 동일하게 맞추었다.

```java
public class JoinRequest {

    private String email;
    private String password;
    private String nickname;
    private String userName;
}
```

```html
<input type="text" name="userName">
```

Service에서 DTO의 값을 Member Entity에 빠짐없이 전달하도록 수정하였다.

```java
Member member = Member.builder()
        .email(request.getEmail())
        .password(encodedPassword)
        .nickname(request.getNickname())
        .userName(request.getUserName())
        .build();
```

---

### ✅ 결과

* 회원가입 시 사용자의 이름이 정상적으로 전달됨
* `USER_NAME` 컬럼의 null 제약조건 오류 해결
* 회원 정보가 DB에 정상적으로 저장됨
* 회원가입 기능 정상 동작 확인

---

### 💡 배운 점

* HTML 입력 필드, DTO, Service, Entity의 필드 연결을 모두 확인해야 한다.
* DB 컬럼에 `NOT NULL` 제약조건이 설정되어 있다면 저장 전에 해당 값이 실제로 Entity에 들어왔는지 확인해야 한다.
* `DataIntegrityViolationException`이 발생하면 DB 제약조건뿐 아니라 DTO와 Entity 사이의 값 전달 과정도 함께 점검해야 한다.

## 2. 이메일 인증 메일이 전송되지 않는 문제

### 📌 문제

회원가입 과정에서 이메일 인증번호 전송 버튼을 눌렀지만 인증 메일이 발송되지 않았고, 메일 서버 연결 오류가 발생하였다.

```text
MailSendException
```

이메일 인증 기능이 동작하지 않아 회원가입 절차를 완료할 수 없었다.

---

### 🔍 원인

`application.yaml`에 작성한 Gmail SMTP 서버 주소에 오타가 있었다.

```yaml
host: stmp.gmail.com
```

SMTP의 올바른 철자는 `smtp`이지만 `stmp`로 잘못 작성하여 애플리케이션이 Gmail 메일 서버에 연결하지 못했다.

또한 Gmail SMTP를 사용하려면 일반 Gmail 비밀번호가 아니라 Google 계정에서 발급한 앱 비밀번호를 사용해야 했다.

---

### 🛠 해결 방법

SMTP 서버 주소를 올바르게 수정하였다.

```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: Gmail 주소
    password: Google 앱 비밀번호
```

TLS 인증과 관련된 설정도 추가하였다.

```yaml
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```

Google 계정의 2단계 인증을 활성화하고 발급받은 앱 비밀번호를 `application.yaml`에 적용하였다.

---

### ✅ 결과

* Spring Boot 애플리케이션과 Gmail SMTP 서버 연결 성공
* 회원가입 이메일 인증번호 정상 발송
* 사용자가 입력한 인증번호 검증 가능
* 이메일 인증 후 회원가입 정상 완료

---

### 💡 배운 점

* 외부 서버 연결 오류가 발생하면 서버 주소, 포트, 인증 방식부터 확인해야 한다.
* 설정 파일의 짧은 오타 하나로도 전체 기능이 동작하지 않을 수 있다.
* Gmail SMTP에서는 일반 계정 비밀번호가 아닌 앱 비밀번호를 사용해야 한다.
* 계정 정보와 앱 비밀번호가 포함된 설정 파일은 GitHub에 공개되지 않도록 관리해야 한다.

## 3. 회원가입·로그인 기능 연동 후 최종 동작 점검

### 📌 문제

회원가입과 로그인 기능을 각각 구현했지만 다음 과정이 하나의 흐름으로 정상 작동하는지 확인이 필요했다.

```text
이메일 인증
→ 회원가입
→ 비밀번호 암호화
→ 로그인
→ 세션 저장
→ Home 페이지 이동
```

각 기능이 개별적으로 작성되어 있어 한 부분에서 값이 누락되면 전체 로그인 흐름이 실패할 가능성이 있었다.

---

### 🔍 점검 내용

다음 항목들을 순서대로 확인하였다.

1. 이메일 인증번호가 정상적으로 발송되는지 확인
2. 사용자가 입력한 인증번호가 세션의 인증번호와 일치하는지 확인
3. 인증이 완료된 이메일만 회원가입할 수 있는지 확인
4. 회원가입 정보가 `MEMBER` 테이블에 정상 저장되는지 확인
5. 비밀번호가 평문이 아닌 암호화된 값으로 저장되는지 확인
6. 로그인 시 `PasswordEncoder.matches()`로 비밀번호를 비교하는지 확인
7. 로그인 성공 후 Member 객체가 세션에 저장되는지 확인
8. 로그인 성공 후 `/home`으로 이동하는지 확인

로그인 성공 시 세션에 회원 정보를 저장하도록 처리하였다.

```java
session.setAttribute("loginMember", member);
```

Home 페이지에서는 로그인 세션이 없을 경우 로그인 페이지로 이동하도록 처리하였다.

```java
@GetMapping("/home")
public String homePage(HttpSession session) {

    Member loginMember =
            (Member) session.getAttribute("loginMember");

    if (loginMember == null) {
        return "redirect:/login";
    }

    return "home";
}
```

---

### ✅ 결과

* 이메일 인증부터 회원가입까지 정상 동작
* 회원가입 정보 DB 저장 성공
* 암호화된 비밀번호를 이용한 로그인 성공
* 로그인 회원 정보가 세션에 정상 저장됨
* 로그인 성공 후 Home 페이지 정상 이동
* 비로그인 상태에서 Home 페이지 접근 시 로그인 페이지로 이동

---

### 💡 배운 점

* 인증 기능은 각 메서드의 동작뿐 아니라 전체 사용자 흐름을 기준으로 테스트해야 한다.
* 로그인 기능에서는 비밀번호 확인뿐 아니라 세션 저장과 페이지 접근 제어까지 함께 구현해야 한다.
* 오류를 해결할 때는 화면, DTO, Service, Entity, DB를 순서대로 확인하면 원인을 찾기 쉽다.

---

## 📝 2026.07.15 트러블슈팅 요약

이날은 회원가입 과정에서 발생한 데이터 누락 문제와 이메일 서버 연결 문제를 해결하였다.

특히 DTO와 Entity의 필드 불일치로 인해 `USER_NAME` 값이 저장되지 않는 문제를 수정하였고, Gmail SMTP 서버 주소 오타와 앱 비밀번호 설정을 바로잡아 이메일 인증 기능을 정상화하였다.

이후 이메일 인증, 회원가입, 로그인, 세션 저장, Home 페이지 이동까지 전체 흐름을 테스트하여 회원 기능이 정상적으로 동작하는 것을 확인하였다.
