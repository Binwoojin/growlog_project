# 📅 Troubleshooting - 2026.07.13

## 1. Spring Boot에서 AWS RDS MySQL 연결 실패

### 📌 문제

GrowLog 프로젝트에서 AWS RDS에 생성한 MySQL 데이터베이스를 Spring Boot와 연결하는 과정에서 데이터베이스에 접속하지 못하는 문제가 발생하였다.

애플리케이션 실행 시 DB 연결 과정에서 통신 오류가 발생하거나, MySQL 사용자 인증 단계에서 연결이 정상적으로 이루어지지 않았다.

```text
JDBCConnectionException
Communications link failure
```

또는 MySQL 인증 방식과 관련된 연결 문제가 발생하였다.

---

### 🔍 원인

Spring Boot와 AWS RDS를 연결하려면 다음 설정이 모두 일치해야 하지만, 초기 구성 과정에서 일부 항목이 누락되거나 올바르게 설정되지 않았다.

* RDS 엔드포인트
* MySQL 포트 `3306`
* 데이터베이스 이름
* 사용자명과 비밀번호
* RDS 퍼블릭 액세스 설정
* 보안 그룹 인바운드 규칙
* JDBC 연결 옵션

또한 MySQL 인증 과정에서 공개키를 가져오는 설정이 허용되지 않아 연결이 실패할 가능성이 있었다.

---

### 🛠 해결 방법

먼저 `application.yaml`의 데이터베이스 연결 정보를 확인하였다.

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://RDS-ENDPOINT:3306/growlog?serverTimezone=Asia/Seoul&characterEncoding=UTF-8&allowPublicKeyRetrieval=true&useSSL=false
    username: DB 사용자명
    password: DB 비밀번호
```

JDBC URL에 다음 옵션을 추가하였다.

```text
allowPublicKeyRetrieval=true
```

AWS RDS 설정에서는 다음 항목을 확인하였다.

```text
퍼블릭 액세스: 허용
포트: 3306
인바운드 규칙: MySQL/Aurora
```

DBeaver에서도 같은 RDS 엔드포인트와 계정으로 접속하여 데이터베이스 연결 상태를 확인하였다.

```sql
USE growlog;

SHOW TABLES;
```

---

### ✅ 결과

* Spring Boot에서 AWS RDS MySQL 연결 성공
* `growlog` 데이터베이스 정상 접근
* JPA를 통해 MEMBER 테이블 사용 가능
* 이후 회원가입 데이터를 RDS에 저장할 수 있는 환경 구성

---

### 💡 배운 점

* `JDBCConnectionException`이 발생하면 Java 코드보다 DB 접속 정보를 먼저 확인해야 한다.
* 원격 데이터베이스는 JDBC 설정뿐 아니라 AWS 보안 그룹과 퍼블릭 액세스 설정도 함께 확인해야 한다.
* 애플리케이션 연결 전에 DBeaver 등의 DB 도구로 먼저 접속을 테스트하면 문제 범위를 좁힐 수 있다.
* 데이터베이스 계정 정보가 들어 있는 설정 파일은 GitHub에 노출되지 않도록 관리해야 한다.

## 2. 비로그인 상태에서도 Home 페이지에 접근할 수 있는 문제

### 📌 문제

초기 Home 페이지는 정적인 HTML, CSS, JavaScript를 중심으로 작성되어 있었으며, `/home` 주소를 직접 입력하면 로그인하지 않은 사용자도 페이지에 접근할 수 있었다.

GrowLog의 Home 페이지는 로그인한 회원의 기록과 목표를 보여주는 회원 전용 화면이므로 로그인 여부를 확인할 필요가 있었다.

---

### 🔍 원인

초기 `HomeController`에서는 `/home` 요청을 받으면 세션 확인 없이 바로 View를 반환하고 있었다.

```java
@GetMapping("/home")
public String homePage() {
    return "home";
}
```

로그인 성공 시 회원 정보를 세션에 저장하더라도, Home 페이지에서 해당 세션의 존재 여부를 확인하지 않으면 URL 직접 접근을 막을 수 없다.

---

### 🛠 해결 방법

`HttpSession`에서 로그인 회원 정보를 가져와 존재 여부를 확인하도록 수정하였다.

```java
@GetMapping("/home")
public String homePage(HttpSession session) {

    Member loginMember =
            (Member) session.getAttribute("loginMember");

    if (loginMember == null) {
        return "redirect:/login";
    }

    return "home";
}
```

로그인 성공 시에는 동일한 세션 키를 사용하여 Member 객체를 저장하도록 구성하였다.

```java
session.setAttribute("loginMember", member);
```

세션에 사용한 키 이름은 다음과 같이 통일하였다.

```text
loginMember
```

---

### ✅ 결과

* 로그인하지 않은 사용자가 `/home`에 접근하면 로그인 페이지로 이동
* 로그인한 사용자만 Home 페이지 이용 가능
* 세션 기반의 기본 접근 제어 적용
* 이후 Goal과 GrowthRecord에도 같은 인증 구조를 적용할 수 있게 됨

---

### 💡 배운 점

* 화면을 단순히 숨기는 것만으로는 회원 전용 페이지를 보호할 수 없다.
* 사용자가 URL을 직접 입력할 수 있으므로 Controller에서 세션을 검증해야 한다.
* 세션을 저장할 때와 조회할 때 동일한 Attribute 이름을 사용해야 한다.
* 회원 전용 Controller마다 같은 코드가 반복되면 이후 인터셉터를 이용한 공통 인증 처리도 고려할 수 있다.

## 3. JSP에서 이미지가 표시되지 않는 문제

### 📌 문제

Home 페이지에 이미지 파일을 추가했지만 브라우저에서 이미지가 정상적으로 표시되지 않는 문제가 발생하였다.

HTML 구조와 파일 자체에는 문제가 없어 보였지만, 실행된 웹 애플리케이션에서 이미지 경로를 찾지 못했다.

---

### 🔍 원인

JSP에서 다음과 같이 루트 기준의 절대 경로를 사용하면 애플리케이션의 Context Path가 반영되지 않을 수 있다.

```html
<img src="/images/growby.png" alt="그로비">
```

Spring Boot 프로젝트가 루트 경로가 아닌 별도의 Context Path로 실행되는 경우 브라우저가 실제 애플리케이션 경로와 다른 위치에서 이미지를 찾게 된다.

또한 이미지가 정적 리소스 경로가 아닌 JSP 폴더 내부에 저장된 경우에도 브라우저가 직접 접근할 수 없다.

---

### 🛠 해결 방법

이미지 파일을 Spring Boot의 정적 리소스 경로에 배치하였다.

```text
src/main/resources/static/images/growby.png
```

JSP에서는 `pageContext.request.contextPath`를 적용하여 현재 애플리케이션의 경로를 기준으로 이미지를 불러오도록 수정하였다.

```jsp
<img
    src="${pageContext.request.contextPath}/images/growby.png"
    alt="그로비">
```

CSS에서 이미지를 불러올 때도 정적 리소스 위치와 URL 경로를 구분하여 확인하였다.

```css
.character-image {
    background-image: url("../images/growby.png");
}
```

---

### ✅ 결과

* Home 페이지에서 이미지 정상 출력
* 애플리케이션 실행 경로가 달라져도 정적 리소스 접근 가능
* JSP와 정적 리소스의 역할 및 경로 구조 정리

---

### 💡 배운 점

* 파일 시스템의 저장 경로와 브라우저가 요청하는 URL 경로는 서로 다르다.
* JSP에서 정적 리소스를 사용할 때는 Context Path를 고려해야 한다.
* 이미지, CSS, JavaScript 파일은 브라우저가 접근할 수 있는 정적 리소스 폴더에 배치해야 한다.
* 이미지가 나오지 않으면 파일명, 확장자, 대소문자, 저장 위치, 요청 URL을 순서대로 확인하는 것이 좋다.

---

## 📝 2026.07.13 트러블슈팅 요약

GrowLog 프로젝트를 시작한 첫날에는 Spring Boot 프로젝트와 AWS RDS MySQL을 연결하고, 로그인 사용자를 기준으로 Home 페이지 접근을 제한하는 기본 구조를 구성하였다.

AWS RDS 연결 과정에서는 엔드포인트, 포트, 보안 그룹, JDBC URL을 점검하고 `allowPublicKeyRetrieval=true` 옵션을 적용하여 데이터베이스 연결 문제를 해결하였다.

또한 `/home`에 직접 접근할 수 있었던 문제를 해결하기 위해 세션의 `loginMember` 존재 여부를 확인하고, 비로그인 사용자는 로그인 페이지로 이동하도록 처리하였다.

JSP 화면에서는 Context Path가 빠져 이미지가 표시되지 않는 문제를 수정하여 정적 리소스 경로를 올바르게 적용하였다.
