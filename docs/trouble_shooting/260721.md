# Troubleshooting (2026-07-21)

## 1. FK 제약조건 수정 (growth_record)

### 문제

`growth_record` 테이블을 생성한 이후 FK 설정을 수정해야 하는 상황이 발생했다.

기존에는 아래와 같이 생성되어 있었지만 이후 요구사항 변경으로 수정이 필요했다.

* Goal 삭제 시 기록은 남겨야 함
* Member 삭제 시 기록은 함께 삭제

### 원인

초기에 FK 옵션을 충분히 고려하지 않고 생성했다.

### 해결

`ALTER TABLE`을 이용하여 FK를 삭제 후 다시 생성하였다.

```sql
ALTER TABLE growth_record
DROP FOREIGN KEY fk_growth_record_goal;

ALTER TABLE growth_record
ADD CONSTRAINT fk_growth_record_goal
FOREIGN KEY (goal_num)
REFERENCES goal(goal_num)
ON UPDATE CASCADE
ON DELETE SET NULL;
```

회원 FK 역시 동일한 방식으로 재설정하였다.

### 결과

* 목표 삭제 시 성장기록은 유지
* 회원 삭제 시 관련 성장기록 삭제

# 3. Goal과 GrowthRecord 관계 설계

### 문제

GrowthRecord가 반드시 Goal과 연결되어야 하는지 고민이 생겼다.

### 원인

일기만 작성하는 기능도 제공하려고 했기 때문이다.

### 해결

ERD를 다시 검토하여

```
goal_num
```

을 Nullable FK로 설계하였다.

즉,

* 목표와 연결된 기록 가능
* 자유로운 일기 작성도 가능

### 결과

두 가지 기능을 모두 지원하는 구조가 되었다.

# 6. Goal 페이지(write) 접근 오류

### 문제

`goal/write` 페이지가 정상적으로 연결되지 않는 문제가 발생하였다.

### 원인

Controller의 매핑, View 경로, 또는 반환 문자열과 실제 JSP 위치가 일치하지 않는 부분을 우선적으로 점검해야 하는 상황이었다.

### 해결 과정

다음 항목들을 순차적으로 확인하였다.

* `@GetMapping("/write")` 존재 여부
* `@RequestMapping("/goal")` 경로 확인
* 반환 View 이름 확인
* `WEB-INF/views/goal/write.jsp` 파일 존재 여부
* JSP 경로와 Controller 반환 문자열 일치 여부

### 결과

매핑과 View 경로를 기준으로 문제를 추적할 수 있는 체크리스트를 정리하였다.

# 7. Git Commit 오류

### 문제

Git Commit 시

```
Git User Name is Not Defined
```

오류가 발생하였다.

### 원인

Git의 사용자 정보(Name, Email)가 설정되어 있지 않았다.

### 해결

```bash
git config --global user.name "사용자명"

git config --global user.email "이메일"
```

설정 후 다시 Commit을 진행하였다.

### 결과

GitHub Commit을 정상적으로 진행할 수 있는 환경이 갖춰졌다.

## 📌 어제 얻은 점

* FK 설계는 삭제 정책(`CASCADE`, `SET NULL`)까지 고려해야 한다.
* Entity와 DB 컬럼명은 정확하게 일치해야 한다.
* GrowthRecord는 Goal에 종속되지 않는 독립적인 기록도 지원하도록 설계하는 것이 확장성이 높다.
* CRUD 기능을 먼저 완성한 뒤 통계(기록 수, 연속 기록 등)를 구현하는 것이 개발 효율이 높다.
* Spring에서는 Controller 매핑과 View 경로가 정확히 일치하는지 먼저 확인하는 습관이 중요하다.
* Git은 프로젝트 시작 시 `user.name`과 `user.email`을 먼저 설정해두면 커밋 오류를 예방할 수 있다.

이 정도로 정리하면 GitHub의 **Troubleshooting** 섹션이나 개발일지에 그대로 올려도 보기 좋은 형태가 될 거야.
