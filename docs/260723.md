# 📅 GrowLog 개발일지 — 2026.07.23

## 1. 오늘의 개발 목표

오늘은 전날 완성한 Goal 및 GrowthRecord 기능의 데이터를 활용하여 GrowLog의 첫 화면인 **Home Dashboard를 완성하는 것**을 주요 목표로 설정하였다.

Home에서는 로그인한 사용자의 현재 활동을 한눈에 확인할 수 있도록 다음 정보를 제공하고자 하였다.

```text
이번 달 성장 기록 수
현재 연속 기록 일수
진행 중인 목표 수
최근 성장 기록
최근 목표
```

Home Dashboard 구현을 완료한 이후에는 다음 개발 기능인 **MyPage와 Attendance 페이지의 연결 구조**도 함께 검토하였다.

---

## 2. Home Dashboard 데이터 구성

기존 `home.jsp`는 HTML, CSS, JavaScript로 화면만 구성되어 있었으며 실제 DB 데이터는 연결되지 않은 상태였다.

오늘은 로그인한 회원을 기준으로 Goal과 GrowthRecord 데이터를 조회하여 Home 화면에 전달하도록 구성하였다.

전체 데이터 흐름은 다음과 같다.

```text
사용자 로그인
→ 세션에서 loginMember 조회
→ HomeController 실행
→ GoalService 및 GrowthRecordService 호출
→ 로그인 회원의 데이터 조회
→ Model에 데이터 저장
→ home.jsp에서 출력
```

Home 화면의 모든 데이터는 현재 로그인한 회원의 `memberNo`를 기준으로 조회하도록 하였다.

---

## 3. HomeController 구조 보완

Home 페이지에서 여러 종류의 데이터를 사용할 수 있도록 `HomeController`에 필요한 Service를 주입하였다.

```java
@Controller
@RequiredArgsConstructor
public class HomeController {

    private final MemberService memberService;
    private final GoalService goalService;
    private final GrowthRecordService growthRecordService;
}
```

Home 페이지 요청이 들어오면 먼저 세션에서 로그인 회원을 확인하도록 하였다.

```java
Member loginMember =
        (Member) session.getAttribute("loginMember");

if (loginMember == null) {
    return "redirect:/login";
}
```

로그인하지 않은 사용자가 `/home` 주소를 직접 입력하더라도 Dashboard에 접근하지 못하도록 기존 세션 검증 구조를 유지하였다.

## 4. 이번 달 성장 기록 수 구현

Home의 요약 카드에서 현재 회원이 이번 달에 작성한 성장 기록 수를 표시하도록 구현하였다.

조회 기준은 현재 날짜가 포함된 월의 첫날과 마지막 날로 설정하였다.

```text
이번 달 시작일
→ 2026-07-01

이번 달 마지막 날
→ 2026-07-31
```

Service에서는 로그인 회원 번호와 날짜 범위를 기준으로 기록 수를 조회하도록 구성하였다.

```text
memberNo
+ 이번 달 시작일
+ 이번 달 마지막 날
→ 이번 달 성장 기록 수
```

조회한 값은 Model에 저장하여 JSP로 전달하였다.

```java
model.addAttribute(
        "monthlyRecordCount",
        monthlyRecordCount
);
```

이를 통해 Home Dashboard에서 사용자가 이번 달에 얼마나 꾸준히 기록했는지 바로 확인할 수 있게 되었다.

---

## 5. 진행 중인 목표 수 연결

현재 로그인한 회원이 등록한 목표 수를 Home의 요약 카드에 표시하였다.

GoalRepository와 GoalService에서 회원 번호를 기준으로 목표 수를 조회하였다.

```java
long goalCount =
        goalService.countGoalsByMember(
                loginMember.getMemberNo()
        );
```

조회한 값은 다음 Model 이름으로 전달하였다.

```java
model.addAttribute("goalCount", goalCount);
```

Home JSP에서는 EL 표현식을 사용하여 목표 개수를 출력하였다.

```jsp
<strong>${goalCount}</strong>
```

이를 통해 사용자는 Home에 접속하자마자 현재 관리 중인 목표 수를 확인할 수 있다.

## 6. 최근 목표 조회 기능 구현

Home Dashboard에는 사용자가 최근 등록한 목표를 최대 3개까지 보여주도록 구성하였다.

Repository에서는 로그인 회원의 목표를 생성일 기준으로 내림차순 정렬하여 최대 3개 조회하도록 하였다.

```java
List<Goal>
findTop3ByMemberMemberNoOrderByCreatedAtDesc(
        Long memberNo
);
```

Service에서는 Home에서 사용할 최근 목표 목록을 반환하였다.

```java
public List<Goal> findRecentGoalsByMember(
        Long memberNo
) {
    return goalRepository
            .findTop3ByMemberMemberNoOrderByCreatedAtDesc(
                    memberNo
            );
}
```

HomeController에서는 조회한 데이터를 Model에 저장하였다.

```java
model.addAttribute(
        "recentGoals",
        recentGoals
);
```

## 7. 최근 목표 카드 출력

`home.jsp`에서는 JSTL의 `<c:forEach>`를 사용하여 최근 목표를 반복 출력하였다.

```jsp
<c:forEach
    items="${recentGoals}"
    var="goalItem"
>
    <!-- 최근 목표 카드 -->
</c:forEach>
```

목표 카드에는 다음 정보를 표시하였다.

* 목표 제목
* 목표 상태
* 목표 진행률
* 진행률 바
* 목표 기간

진행률은 숫자와 막대 형태로 함께 표시하였다.

```jsp
<span>${goalItem.goalProgress}%</span>

<div class="progress-bar">
    <div
        class="progress-value"
        style="width: ${goalItem.goalProgress}%;">
    </div>
</div>
```

이를 통해 목표의 현재 진행 상황을 시각적으로 확인할 수 있도록 하였다.

## 8. 최근 성장 기록 조회 기능 구현

Home에서는 사용자가 최근에 작성한 성장 기록도 확인할 수 있도록 구성하였다.

최근 기록은 작성일 또는 생성일 기준으로 정렬하고, Home 화면에 필요한 개수만 조회하도록 하였다.

조회 조건은 다음과 같다.

```text
로그인 회원의 기록
→ 최신순 정렬
→ Home에 필요한 개수만 조회
```

Controller에서는 조회한 최근 기록 목록을 Model에 저장하였다.

```java
model.addAttribute(
        "recentRecords",
        recentRecords
);
```

## 9. 최근 성장 기록 카드 출력

`home.jsp`에서는 최근 성장 기록을 반복 출력하였다.

각 카드에는 다음 정보를 표시하도록 구성하였다.

* 성장 기록 제목
* 기록 내용 일부
* 작성 날짜
* 연결된 목표 또는 자유 기록 여부
* 난이도 또는 오늘의 기분
* 상세 페이지 이동 링크

성장 기록을 클릭하면 해당 기록의 상세 페이지로 이동하도록 연결하였다.

```jsp
<a href="${pageContext.request.contextPath}/record/${recordItem.recordNum}">
    <!-- 최근 성장 기록 내용 -->
</a>
```

기록 내용이 길더라도 Home 카드 영역을 벗어나지 않도록 일부 내용만 노출하는 UI를 적용하였다.

## 10. 연속 기록 카드 구성

사용자의 꾸준한 기록 활동을 보여주기 위해 Home에 **연속 기록 카드**를 구성하였다.

연속 기록 기능은 다음 기준을 바탕으로 계산할 수 있도록 구조를 검토하였다.

```text
오늘 성장 기록이 존재하는가?
→ 존재하면 오늘을 포함하여 연속 일수 계산

오늘 기록이 존재하지 않는가?
→ 어제부터 과거 방향으로 연속 일수 계산
```

Home에서는 현재 연속 기록 일수를 보여주고, 카드를 선택하면 추후 구현할 Attendance 페이지로 이동할 수 있도록 설계하였다.

## 11. Home 출석 카드 링크 적용

기존 연속 출석 카드는 `<article>` 태그로만 구성되어 있어 클릭해도 페이지 이동이 되지 않았다.

카드 전체를 클릭할 수 있도록 `<a>` 태그를 바깥쪽에 배치하는 구조를 검토하였다.

```jsp
<a
    href="${pageContext.request.contextPath}/mypage/attendance"
    class="summary-card-link"
>
    <article class="summary-card">
        <!-- 연속 출석 정보 -->
    </article>
</a>
```

이 구조를 사용하면 카드 내부의 디자인은 유지하면서 전체 카드가 Attendance 페이지로 이동하는 링크 역할을 할 수 있다.

링크에 기본적으로 적용되는 밑줄과 글자 색상을 제거하기 위해 CSS도 함께 구성할 수 있다.

```css
.summary-card-link {
    display: block;
    color: inherit;
    text-decoration: none;
}
```

## 12. 빈 데이터 화면 처리

새로 가입한 회원은 아직 목표나 성장 기록이 없을 수 있으므로 빈 목록에 대한 화면 처리도 함께 구성하였다.

### 최근 목표가 없는 경우

```jsp
<c:if test="${empty recentGoals}">
    <p>아직 등록된 목표가 없습니다.</p>
</c:if>
```

### 최근 성장 기록이 없는 경우

```jsp
<c:if test="${empty recentRecords}">
    <p>아직 작성한 성장 기록이 없습니다.</p>
</c:if>
```

단순한 빈 공간으로 표시하지 않고 사용자가 다음 행동을 할 수 있도록 목표 등록 또는 기록 작성 페이지로 이동하는 안내 버튼을 제공하는 방향도 검토하였다.

## 15. MyPage와 Attendance의 관계 설계

Attendance 페이지는 독립적인 메인 메뉴보다 사용자의 개인 활동을 확인하는 기능에 가깝기 때문에 **MyPage의 하위 메뉴**로 구성하는 것이 적절하다고 판단하였다.

MyPage의 주요 메뉴는 다음과 같이 설계하였다.

```text
MyPage
├─ 프로필 정보
├─ 계정 설정
├─ 개인정보 수정
├─ 비밀번호 변경
└─ 출석 기록
```

Attendance는 다음과 같은 URL 구조를 사용하는 방향으로 검토하였다.

```text
/mypage
→ 마이페이지 메인

/mypage/profile
→ 개인정보 및 프로필 수정

/mypage/account
→ 계정 설정

/mypage/attendance
→ 출석 기록
```

이를 통해 MyPage 내부에서 일관된 사이드 메뉴 또는 탭 구조를 사용할 수 있다.

## 16. MyPage Controller 구성 방향

MyPage와 Attendance 페이지를 연결하기 위해 공통 Controller 구조를 검토하였다.

```java
@Controller
@RequiredArgsConstructor
@RequestMapping("/mypage")
public class MyPageController {

    @GetMapping
    public String myPage(
            HttpSession session,
            Model model
    ) {
        // 로그인 세션 확인
        // 회원 정보 조회
        // 마이페이지 메인 화면 반환

        return "mypage/index";
    }

    @GetMapping("/attendance")
    public String attendancePage(
            HttpSession session,
            Model model
    ) {
        // 로그인 세션 확인
        // 연속 출석 및 월별 출석 정보 조회
        // Attendance 화면 반환

        return "mypage/attendance";
    }
}
```

MyPage와 Attendance는 모두 회원 개인 데이터에 접근하므로 각 요청에서 로그인 세션을 먼저 확인해야 한다.

## 17. MyPage 화면 구조 검토

MyPage의 개인 설정 기능과 Attendance 페이지가 시각적으로 분리되지 않도록 공통 레이아웃을 사용하는 방향으로 설계하였다.

예정된 화면 구조는 다음과 같다.

```text
MyPage 공통 헤더
┌──────────────────────────────┐
│ 사이드 메뉴 │ 페이지 콘텐츠 │
│             │               │
│ 프로필      │ 선택한 메뉴    │
│ 계정 설정   │ 상세 내용      │
│ 출석 기록   │               │
└──────────────────────────────┘
```

Attendance를 MyPage의 하위 콘텐츠로 구성하면 사용자가 개인 정보와 활동 기록을 한 공간에서 관리할 수 있다.

## 18. 오늘 발생한 주요 확인 사항

### 최근 데이터가 화면에 표시되지 않는 문제 점검

Home에서 최근 목표 또는 성장 기록이 표시되지 않을 경우 다음 항목을 확인하였다.

```text
Repository 조회 메서드
→ Service 반환값
→ Controller의 Model 이름
→ JSP의 items 속성
→ 반복문의 var 이름
```

Controller에서 전달한 Model 이름과 JSP에서 참조하는 이름이 일치해야 한다.

```java
model.addAttribute("recentGoals", recentGoals);
```

```jsp
<c:forEach
    items="${recentGoals}"
    var="goalItem"
>
```

또한 반복문에서 선언한 변수명과 내부에서 사용하는 변수명이 같아야 한다.

```jsp
${goalItem.goalTitle}
```

### 출석 카드에 링크가 없는 문제

`<article>`은 콘텐츠 영역을 표현하는 태그이지만 페이지 이동 기능은 없으므로 `<a>` 태그를 사용하여 Attendance URL과 연결하는 방식으로 정리하였다.

## 21. 오늘 배운 점

* Dashboard는 여러 기능의 데이터를 모아 보여주기 때문에 Controller에서 각 Service의 역할을 명확히 구분해야 한다.
* Home에 표시되는 데이터도 반드시 로그인한 회원 번호를 기준으로 조회해야 한다.
* Repository에서 필요한 개수만 조회하면 불필요한 전체 데이터를 가져오지 않아도 된다.
* JSTL 반복문에서는 Model 이름, `items`, `var`, 내부 참조 이름을 일치시켜야 한다.
* 카드 전체에 이동 기능을 적용하려면 단순한 `<article>`이 아니라 `<a>` 태그와 함께 구성해야 한다.
* 연속 출석 기능은 단순한 카운트가 아니라 오늘 기록 여부와 날짜의 연속성을 함께 고려해야 한다.
* Attendance는 사용자 개인 활동 데이터이므로 MyPage의 하위 메뉴로 구성하면 서비스 구조가 자연스럽다.
* 다음 기능을 개발하기 전에 URL과 화면 계층 구조를 먼저 정리하면 Controller와 JSP를 일관성 있게 구성할 수 있다.
